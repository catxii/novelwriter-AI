# NovelWriter

NovelWriter 是一款专为原创小说与长篇内容创作设计的 AI 写作工作台。它将现代代码编辑器的高效交互体验与传统写作软件的沉浸式创作环境相结合，帮助作者完成从灵感构思到章节成稿的完整创作流程。

项目支持角色设定、世界观构建、剧情规划、章节大纲生成、正文续写以及长期记忆管理等功能。通过集成多种大语言模型接口，NovelWriter 能够根据作品设定和上下文内容提供持续、连贯的创作辅助，帮助作者保持故事一致性和创作效率。

与传统 AI 聊天工具不同，NovelWriter 更关注长篇叙事场景下的创作需求。系统通过长期记忆、项目资料管理和结构化写作流程，使 AI 能够更好地理解角色关系、世界观规则和剧情发展脉络，从而为小说创作提供更加稳定和可控的辅助能力。

无论是网络小说作者、独立创作者，还是探索 AI 创意写作的开发者，NovelWriter 都可以作为一个开放、可扩展的创作平台，为长篇内容生产提供支持。

## 运行

```bash
npm install
npm run dev
```

## 接入 Ollama

1. 本机启动 Ollama。
2. 拉取一个中文写作可用模型，例如：

```bash
ollama pull qwen3:8b
```

3. 在右侧「接口」面板中选择 `Ollama`。
4. Base URL 使用 `http://localhost:11434`。
5. 模型填写本机已有模型名。当前项目默认值是这台机器上已存在的 `gemma4:e2b`。
6. 点击「测试连接」。

## 接入其他 API

右侧「接口」面板选择 `OpenAI-compatible`，填写：

- Base URL：例如 `https://api.example.com/v1`
- 模型：服务商提供的模型名
- API Key：服务商密钥

应用会请求 `POST /chat/completions`，适合大多数兼容 OpenAI Chat Completions 协议的服务。

## 快捷动作

- `1` 续写：按当前文风继续写。
- `2` 润色：保留剧情，强化画面和节奏。
- `3` 扩写：把片段扩成更完整的一场戏。
- `4` 记忆：抽取人物、世界观、伏笔等长期记忆。
- `+` 自定义：执行右侧接口面板里的自定义指令。

正文、记忆和接口配置会保存在浏览器本地存储中。
